    (*) 직접 컴파일

1. Voyage Linux

제가 Real Time 실시간 커널을 좋아하지 않아서 일반 커널의 Voyage Linux를 선택했습니다.
데비안 Jessie 버전인데, 개발자가 최신 Stretch 지원은 계획이 없다고 합니다.
패키지를 업데이트하면 Voyage 저장소의 공개키 오류로 진행이 되지 않습니다.
따라서 Voyage 패키지는 사용하지 않고 데비안 패키지 만으로 설정했습니다.

2. USB 자동 마운트

포고 렌더러와 같이 USB 자동 마운트를 지원합니다.

다만 알릭스의 시스템이 읽기 전용이라, 다음 방법을 쓰시면 됩니다.

한 번만 알릭스에 SSH로 로그인해서 'remountrw' 명령어를 실행한 다음 USB 장치를 연결합니다.

reboot이나 poweroff로 정상적으로 시스템을 다시 시작하면 이후부터는 해당 USB를 자동으로 인식할 겁니다.

그런 다음 MPD 클라이언트에서 DB를 업데이트하면 연결된 USB의 음원이 보입니다.

3. CIFS와 NFS 지원

cifs-utils와 nfs-common 패키지를 설치하여 CIFS와 NFS를 지원합니다.
이로써 프로세스가 추가되기는 하는데, 기존 방식으로 사용하시는 분들도 있을 것 같아서요.
PC나 NAS 음원을 직접 마운트해서 사용하시려면 /etc/fstab 파일을 수정하면 됩니다.


4. PCM 디코더

MPD를 가볍게 하려고 PCM 디코더를 사용하지 않도록 설정해 두었습니다.
푸바의 UPnP MediaRenderer Output 등 일부 PCM 디코더 출력에 문제가 있을 수 있습니다.

아래 명령어로 다시 시작해서 MPD의 PCM 디코더를 사용할 수 있습니다.

 # remountrw

 # sed -i '/plugin.*pcm/!b;n;s/no/yes/' /etc/mpd.conf.sav

제 기억으로는 디코딩된 PCM보다 원본 파일을 렌더러로 바로 전달하는 것이 듣기에 더 좋았던 것 같습니다.


5. DSD 지원

기존 렌더러와 마찬가지로 DoP(DSD over PCM)로 기본 설정해 두었습니다.
Native DSD를 시험해 보시려면, Putty 등으로 SSH로 연결해서 아래 명령어로 다시 시작해 보시기 바랍니다.


 # remountrw                                 # 읽고 쓰기로 전환
 # sed -i /dop/s/^/#/' /etc/mpd.conf.sav     # DoP 사용하지 않음 


Native DSD에서 DoP로 바꾸는 것은 아래 명령어입니다.

 # remountrw                                 # 읽고 쓰기로 전환
 # sed -i /dop/s/^#*//' /etc/mpd.conf.sav    # DoP 사용함

6. 기타 튜닝

기존 렌더러와 비슷한 경향으로 튜닝을 하였습니다.
들으시는 취향에 맞을 지는 모르겠습니다.

이외에 대부분 사항은 기존 렌더러와 비슷하므로 이전 글들을 참조하시기 바랍니다.

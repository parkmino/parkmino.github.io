당분간 마지막 업데이트라 MPD 최신 버전을 컴파일하려고 했는데, armel에서 소스 코드로는 곧바로 빌드가 안된다고 하더군요.

차선책으로 데비안 백포트 버전으로 0.19.12로 업데이트하였습니다.


MPD를 프로세스 전면에 배치하고 우선권을 줄여서 보다 차분하면서도 자연스럽게 들릴 겁니다.

이에 따라 커널 스케줄러도 미세하게 조정했습니다.

Upmpdcli 배경에 nohup을 적용하여 약간이나마 부하를 줄였습니다.

Ape 파일 재생이 안되는 버그로 Upmpdcli에 관련 파일 포맷을 추가했습니다. (다낭님)

그리고 USB DAC 사운드와 ALSA 카드를 모두 '0'으로 설정하여, 일부 USB DDC에서 출력이 안되는 문제를 해결하였습니다.


USB 자동 마운트를 지원합니다.

Udev rules를 사용하여 읽기 전용으로 연결하는데, 혹시 잘 안되면 USB를 다시 연결해 보시기 바랍니다.

USB 장치를 연결하고 MPD 클라이언트에서 DB 업데이트를 하면 음원이 보일 겁니다.

포고 E02를 기준으로 첫 번째 부팅 USB를 제외하고 두 번째 USB부터 자동 마운트 합니다.

포고 모바일에서 SD로 부팅한다면 첫 번째 USB부터 마운트 하도록 아래처럼 수정할 수 있습니다.

 # sed -i 's/sd\[b-z\]/sd\[a-z\]/g' /lib/udev/rules.d/11-my-media-automount.rules

이 기능을 사용하지 않으려면 아래 명령어를 참조하시기 바랍니다.

 # sed -i 's/^/#/' /lib/udev/rules.d/11-my-media-automount.rules


【참고】 사용법


데비안용으로 개조된 부트로더에서 8GB 용량 이상의 USB나 SD카드로 부팅할 수 있습니다.

파일 시스템은 일부 기기와의 호환성 문제로 ext3이 아닌 ext2로 만들었습니다.

전면 로고 LED가 커지면 제대로 부팅이 완료된 것입니다.

SSH 연결이 가능하며 아이디와 패스워드는 모두 'root'입니다.

스와프 파티션이나 파일 없이 주어진 메모리에서만 동작합니다.


오디오 렌더러는 부팅과 함께 바로 사용 가능합니다.

외부 음원의 마운트는 'sudo nano /etc/fstab'으로 수정하고, 'sudo reboot'으로 재시작하고, 'mpc update'를 실행하면 음원이 보일 겁니다.

이외에 ALSA 사운드 확인이나 MPD 출력을 설정할 필요 없이 곧바로 재생하면 됩니다.

참고로, ALSA 사운드를 확인하려면 'aplay -l' 대신에 'cat /proc/asound/modules' 또는 'cat /proc/asound/card*/pcm*p/info'를 실행하면 됩니다.


【참고】 부팅이 안 될 때 (ext2 파일 시스템)


이미지를 만들고 첫 번째 부팅이 안되면 잠시 후에 다시 시도해 보시기 바랍니다.

그래도 부팅이 안되면 파일 시스템에 문제가 있을 수 있습니다.

일부 기종에서의 호환성 문제로 ext2 파일 시스템으로 만들었습니다.

시스템 환경의 루트 파일 시스템이 ext3/4로 설정되어 있다면 부팅이 안 될 수도 있습니다.


부팅이 제대로 안된다면 기존 부트로더로 부팅한 다음 아래 명령어를 실행해 보시기 바랍니다.

 # /tmp/fw_printenv                     # 환경 변수 확인

 # /tmp/fw_setenv usb_rootfstype ext2

 # /tmp/fw_setenv rootfstype ext2


USB to TTL 시리얼 통신의 경우 'setenv'로 수정하고 'saveenv'으로 저장하면 된다고 합니다. (투모로우님)


1. 포고플러그 하드웨어 사양


   

  ○ 포고 E02

    - 프로세스 : Marvell Kirkwood 1.2GHz 싱글코어 (ARMv5te)

    - 메모리 : 256MB

    - NAND : 128MB

    - USB : 2.0 (4개)

    - 이더넷 : 기가 비트


    

  ○ 포고 모바일 (V4)

    - 프로세스 : Marvell Kirkwood 800Hz 싱글코어 (ARMv5te)

    - 메모리 : 128MB

    - NAND : 128MB

    - SD : 1개

    - USB : 2.0 (1개)

    - 이더넷 : 기가 비트


  ○ 포고 시리즈 4 (V4)

    - 포고 모바일 + SD + SATA + USB 3.0 2개.


사양은 싱글코어 CPU에 모델에 따라 128/256MB 메모리로 전반적으로 낮은 편입니다.

가능하면 미니멀 배포판에 꼭 필요한 서비스만 설치하는 것이 유리할 겁니다.

오디오 렌더러로 사용하기에 무리는 없는데, 싱글코어라 트윅에 한계가 있기는 합니다.

개조는 먼저 U-Boot 부트로더를 수정해야 하는데 데비안용은 아래에 있습니다.

최신이 아니더라도 개조된 기존 부트로더에서도 최신 데비안을 설치할 수 있습니다.

참고로 제가 받은 포고 E02의 부트로더는 'U-Boot 2011.12 (Feb 20 2012 - 21:21:59)'로 구형입니다.

번거롭지만 USB to TTL 시리얼 케이블로 직접 연결하여 수정할 수는 있다고 하니 참조하시기 바랍니다.

4.4.0 리눅스 커널 기반의 armel 아키텍처로 상당히 가벼워서 좋네요.

다른 리눅스에서 설치해야 하는데, 저는 우분투 노트북에서 설치하였습니다.


참고로 데비안에서 지원하는 ARM 아키텍처는 구형(armel), 신형(armhf), 64비트(arm64)가 있습니다.

포고는 구형 armel, 라즈베리 파이 2는 armhf, 라즈베리 파이 3은 arm64까지 지원이 가능합니다.

따라서 라즈베리 파이와 포고플러그는 바이너리 호환이 안됩니다.

Upmpdcli는 개발자가 armhf 패키지만 지원하므로 사용자가 직접 armel로 빌드 해야 합니다.


3. 오디오 렌더러


MPD는 데비안 순정 0.19.1 버전을 설치하고, Upmpdcli는 1.2.12 최신 버전을 직접 빌드 했습니다.

NFS 서버/클라이언트와 MPD의 DoP를 기본 지원합니다.

라즈베리 파이에 비해서는 부팅 시간이 좀 길어서 한참(?)을 기다려야 합니다.

기존 라즈베리 파이용 오디오 렌더러를 기반으로 만들었으므로 이전 블로그 글을 참조하시기 바랍니다.

다만 메뉴나 USB 트윅 등 상세한 설정은 지원하지 않습니다.


4. 설정과 업데이트


최초 부팅 시 개발자가 권장하는 SSH 보안 관련 내용입니다.

 # rm /etc/ssh/ssh_host*

 # ssh-keygen -A


정확한 원인은 모르겠지만 업데이트는 ext3 파일 시스템에서는 문제가 없으나, ext2에서는 재시작이 되지 않더군요.

ext2나 이 기반인 오디오 렌더러에서는 업데이트를 하지 않는 것이 좋겠습니다.

참고로, ext3에서의 업데이트는 SSH로 연결해서 아래 명령어를 차례로 입력하면 됩니다.

 # apt-get update

 # apt-get upgrade

 # apt-get dist-upgrade

 # apt-get autoremove


CIFS로 윈도 장치와도 연결하려면 아래 패키지를 설치합니다.

그런 다음 /etc/fstab을 수정하시면 됩니다. (...cifs...)

 # apt-get install cifs-utils


오디오 렌더러라서 메타데이터를 사용하지 않도록 설정했습니다.

음원 태그를 읽지 않는 장점은 음원 관리가 빠르다는 거고, 단점은 음원이 파일명으로 보이고 음반 표지가 안 보인다는 겁니다.

마운트한 음원에서 태그를 사용하려면 아래 명령어를 입력합니다.

그런 다음 시스템을 재시작하고 MPD 클라이언트에서 음원 DB를 다시 읽으면 됩니다.

 # sed -i '/metadata.*artist/s/^#*//; /metadata.*none/s/^/#/' /etc/mpd.conf

 # reboot


MPD와 오디오 렌더러(Upmpdcli)를 사용하지 않고 다른 용도로 쓰려면 아래 명령어를 참조하면 됩니다.

 # cp /etc/rc.local /etc/rc.local.audio             # 파일 백업

 # sed -i '13,${/^exit 0/!d}; $i\\' /etc/rc.local   # 파일 내용 지우기
